# gates-review：季检

**功能一句话**：凡在 buffett-gates 通过检验、进入视野的公司，每个季度做一次检查——对**事实、价格、认知**三个层面读位移、做归因。

范式是**体检，不是审判**：巴菲特买入后每年读年报，读的不是"我的理论对不对"，而是两个读数动了没——生意的经济特征变了吗（1996 信：买入后要做的就是确认经济特征没有变化），市场先生现在报什么价。**没变化就是好消息，读数动了才查原因。**

## 输入

该标的最近一份快照，须含 PROTOCOL 两块接口（格式见仓库根 PROTOCOL.md）：

- **仪表读数记录**（四闸读数 + 象限坐标 + 当时价格 + 数据截止日）——位移差分的锚点
- **退出信号表**——底线校验的依据

外来快照（其他框架的报告、手写笔记）若缺仪表读数记录：首次运行先按 buffett-gates 三闸补一份基线读数，从第二次季检起才有位移与轨迹。

## 执行

### 1. 增量采集

只采上次检查日以来的新东西：新披露财报、公告、区间行情、重大行业事件。不全量重采。

数据硬规则与 buffett-gates 数据链相同：最新报告期 / 来源标注 / 市值手工校验 / 状态事实以披露平台为准。

### 2. 三层位移与归因（核心动作）

| 层 | 问什么 | 对应仪表 |
|---|---|---|
| **事实** | 生意本身变了吗——经济特征、竞争格局、财报数据 | 锚 + 竞争闸读数更新 |
| **价格** | 市场先生的报价变了吗——折扣扩大还是收窄 | 情绪闸读数更新 |
| **认知** | 我上季的判断还站得住吗——有没有看错的地方 | 认知闸读数更新；**这层最容易被藏起来，强制作答，不许空** |

每层一行：上期读数 → 本期读数 → 位移方向 → 一句归因。归因强制三选一：**Δ事实 / Δ价格 / Δ认知**。

三层皆无位移 → 一句话收工：**"无变化，好消息。"**

### 3. 输出四件

1. **三层位移表**（上期读数 → 本期读数 → 归因）
2. **四象限轨迹**：仪表盘图上标三点——基线点（首次快照）→ 上期点 → 本期点，连成轨迹，看它往哪个格子挪
3. **退出信号底线校验**：逐条对照**基线**退出信号表算累计位移——相邻差分抓本季新东西，累计校验防"每季位移都小、四季出了格子"的温水漂移
4. **判定一句话**：持有 / 加注 / 减仓 / 退出 / 格子迁移警告

季检报告末尾附**本期仪表读数记录**（PROTOCOL 格式），作为下季差分的锚点。

### 4. 防锚定（内置规则，不设独立模式）

当归因出现 **Δ认知**、或**轨迹跨格**时，追加一次**新生之眼**：不看旧快照，独立重跑一遍 buffett-gates，两份读数对照——轨迹说"它挪到哪了"，新生之眼说"没看过轨迹的人会把它标在哪"，夹角大 = 有一边在自欺。平静季度不跑，省算力。

## 节奏

每季财报披露后跑一次（A 股节奏：一季报/年报 ≤4/30、半年报 ≤8/31、三季报 ≤10/31；其他市场随当地披露节奏）。可配定时任务自动触发。

输出目录：用户当前目录，或由用户参数指定。
