# buffett-gates：验金台

**只做一件事**：标的从哪来不管——自己选的、筛选工具出的、券商研报推的——放上来，验成色：**这个价格、以我的认知，配不配下注；下注之后，赌注对不对。**

不筛选、不做全流程投研。理论地基见 [references/rank.md](references/rank.md)（一锚三闸降秩全文）；每个概念的原文出处见 [references/cards/](references/cards/)（10 张概念卡，按需加载，每张含股东信原文引文 + 收口检验问题）。

## 两个阶段

**投资阶段**（产出快照——回答"这个价格、以我的认知，配不配下注"）：输入标的（名称/代码）→ 域检 → 法眼 → 数据链 → 三闸 → 四象限 → 快照

**持有阶段·季检**（读投资阶段产出的快照——回答"当初的判断，现在还站不站得住"）：输入该标的最近一份快照（须含 PROTOCOL 两块接口：仪表读数记录 + 退出信号表；来源不限，哪怕不是本 skill 自己产出的，只要格式对得上）→ 增量采集 → 三层位移与归因（事实/价格/认知）→ 四象限轨迹 → 退出信号底线校验 → 判定

## 流程（投资阶段：产出快照）

### 1 域检（门槛，先于一切）

只验上市公司：**有活跃市场报价 + 有定期披露义务**，两者缺一即域外。原因（此处说明一次，报告中不重复）：情绪闸靠市场先生的报价、安全边际靠折扣计算——无报价则四象限缺一根轴，强行跑就是硬填表。pre-IPO、摘牌股、流动性枯竭的挂牌股，都挡在这里。

报告中的写法：
- **通过 → 只写一行**（如"域检：上交所主板，报价与披露完整，放行"），不阐述理由
- **域外 → 短段说明**：它缺什么、等什么条件再来验（"域外，等它上市/恢复活跃报价再来验"），然后停笔——理由只在出局时才值得占篇幅

### 2 法眼（准入判定）

过了域检，再验一眼：**这标的在巴菲特的定义域里吗？** 两问，占其一即"相在"：

- **好生意之相**：它像不像一台能长期产出现金的机器？（有没有让人想往下验的经济特征——异常的高回报、异常的粘性、异常的定价权）
- **好错价之相**：市场先生对它的报价像不像犯了病？（无论生意平庸与否，价格与价值的裂口大到值得看一眼）

二选一，不许骑墙：

- **相在** → 一句话点出相在哪，发准入证，进数据链
- **两相皆无** → 一句话说清它既不是印钞机、也没被错价——**当场停笔，整份报告就这一段。** 不值得验的标的零成本出局。

### 3 数据链

数据分两类，采集方法不同——更新频率不同，工具选型也该不同：

- **行情类**（股价、市值——高频，每天甚至每分钟变）：直接 WebFetch 行情接口或个股行情页面拿当前显示价（如新浪行情接口 `hq.sinajs.cn/list=<市场前缀><代码>`、腾讯行情接口 `qt.gtimg.cn/q=<市场前缀><代码>`，或财经网站个股行情页当前价）。**不用 WebSearch"搜"**——搜索引擎索引的是媒体发过的历史资讯，天然滞后，搜不到"今天"的价格是正常的，不代表数据不存在，是工具选错了
- **财务/公告类**（营收、净利润、股本等——低频，按季更新）：采集顺序为交易所/监管披露平台公告原文（最终核实依据，**优先找 HTML 摘要版，不是 PDF 附件**——定期报告摘要常有媒体转载的 HTML 版本，PDF 附件解析失败率高）→ akshare（如已安装，结构化数据）→ WebSearch（定位线索，不直接采信媒体转述的数字）→ 要求用户提供数据（前面都拿不到时，明确告知缺口，不编造、不用训练知识里的旧数字顶替）

硬规则（每条都有事故来源，不可省）：

1. **标的身份先核实**：公司名称/代码对应的准确交易所代码与上市地先核实清楚（防同名混淆），这是后续所有数据的锚点
2. **最新报告期，主动反推**：用当前日期反推应披露到哪个报告期（A 股节奏：一季报/年报 ≤4/30、半年报 ≤8/31、三季报 ≤10/31；其他市场按当地节奏），主动核对是否已有比训练知识更新的定期报告或临时公告，不停在训练截止时间
3. **股本核实**：股本/市值须核实公告原文；市值 = 股价 × 总股本手工校验，禁止心算
4. **来源标注到可复核程度**：数据逐项标注来源与发布日期，标到具体公告/文件，不能只写"某研报"或"网上查到"；WebSearch 搜到的媒体转述只是线索，要顺着找到交易所/公司公告原文才算数
5. **多源不一致，以一手披露为准**：不同信息源给出的同一数字若对不上，以交易所/公司披露平台原文裁决，不挑一个看着顺眼的用
6. **信息丰富度分级**：A 数据充裕 / B 需推算（标注置信度）/ C 极度稀缺——C 级不硬填表，诚实降读数并标注"数据不足无法判定"；行情类拿不到当前价时，如实标注价格的实际读数日期，不用旧价冒充当天
7. **状态事实核验**：挂牌/摘牌/停复牌等公司状态，以交易所/披露平台一手信息为准，搜索引擎旧闻不作数

### 4 三闸逐一读数

每闸：判定问题 + 原文依据年份，一句话读数，不许骑墙。原文全文见对应概念卡。

- **锚（估值地基）**：伊索三问答得出吗——丛里有鸟？何时出来几只？无风险利率多少？owner earnings（报告利润 + 折旧摊销 − 维持性资本开支）可估吗？答不出 → 本标的属投机，直接出局，后面不用看。（1986/2000 信 → [cards/anchor-owner-earnings.md](references/cards/anchor-owner-earnings.md)）
- **竞争闸（余生长度，确定性读数上半）**：十年后护城河更宽还是更窄，凭什么（低成本/品牌/网络效应/转换成本）？护城河需不断重建的快变行业 → 读数直接给低，无论多便宜。Mayo Clinic 测试：遮住 CEO 名字还是好生意吗？（2007 信 → [cards/gate-moat.md](references/cards/gate-moat.md)）
- **认知闸（域 + 量，确定性读数下半 + 折扣要求）**：域——这个判断在能力圈内还是圈边？能向聪明外行讲清它十年后怎么赚钱吗？圈边缘按圈外处理。量——保守估算下价格比价值低多少？只是略低 → 不买。（1996/1992 信 → [cards/gate-circle.md](references/cards/gate-circle.md)、[cards/gate-margin.md](references/cards/gate-margin.md)）
- **情绪闸（折扣读数）**：当前报价是市场先生亢奋价还是抑郁价？买入理由是"价值高于价格"还是"价格在涨"？明天跌 50% 且五年不能卖，敢加仓吗？（1987 信 → [cards/gate-mr-market.md](references/cards/gate-mr-market.md)）

注：本流程不设管理层闸——巴菲特体系的原点之一是不接触公司、仅凭公开披露即可决策，域检已保证输入全是披露充分的上市公司。管理层视角的参考资料见 [cards/derived-institutional-imperative.md](references/cards/derived-institutional-imperative.md)。

### 5 四象限定格

纵轴 **确定性** = 竞争闸 × 认知闸(域)；横轴 **折扣** = 情绪闸 × 认知闸(量)。两轴独立：确定性由企业和我决定，折扣由市场先生一个人决定。

| 格子 | 判定 | 动作 |
|---|---|---|
| 黄金格（左上） | 高确定 + 大折扣 | 可重注；退出只看护城河与高估，不看股价 |
| 等待格（右上） | 高确定 + 无折扣 | 趴着，入观察清单，等市场先生抑郁 |
| 烟蒂格（左下） | 低确定 + 大折扣 | 小仓或放弃；进则价差兑现即走 |
| 投机格（右下） | 低确定 + 无折扣 | 永不 |

**格子决定动作；格子结论必须与最终建议一致——不一致说明有一边在自欺，回头重判。**

### 7 输出：快照报告

必含两块接口（格式规范见仓库根 PROTOCOL.md，这是投资阶段与持有阶段之间的唯一衔接）：

1. **仪表读数记录**：四闸读数（10 格刻度）+ 四象限坐标（格子 + 格内方位）+ 当时价格 + 数据截止日——持有阶段做位移差分的锚点
2. **退出信号表**：`| # | 假设 | 证伪信号 | 检查周期 |`——3–6 条，证伪信号必须是可观测的硬数据条件

输出目录：用户当前目录，或由用户参数指定。

## 季检（持有阶段：每季度）

**功能一句话**：凡在投资阶段通过检验、进入视野的公司，每个季度做一次检查——对**事实、价格、认知**三个层面读位移、做归因。

范式是**体检，不是审判**：巴菲特买入后每年读年报，读的不是"我的理论对不对"，而是两个读数动了没——生意的经济特征变了吗（1996 信：买入后要做的就是确认经济特征没有变化），市场先生现在报什么价。**没变化就是好消息，读数动了才查原因。**

### 输入

该标的最近一份快照，须含 PROTOCOL 两块接口（格式见仓库根 PROTOCOL.md）：

- **仪表读数记录**（四闸读数 + 象限坐标 + 当时价格 + 数据截止日）——位移差分的锚点
- **退出信号表**——底线校验的依据

外来快照（其他框架的报告、手写笔记）若缺仪表读数记录：首次运行先跑投资阶段流程补一份基线读数，从第二次季检起才有位移与轨迹。

### 增量采集

只采上次检查日以来的新东西：新披露财报、公告、区间行情、重大行业事件。不全量重采。

数据硬规则与第 3 节数据链相同：最新报告期 / 来源标注 / 市值手工校验 / 状态事实以披露平台为准。

### 三层位移与归因（核心动作）

| 层 | 问什么 | 对应仪表 |
|---|---|---|
| **事实** | 生意本身变了吗——经济特征、竞争格局、财报数据 | 锚 + 竞争闸读数更新 |
| **价格** | 市场先生的报价变了吗——折扣扩大还是收窄 | 情绪闸读数更新 |
| **认知** | 我上季的判断还站得住吗——有没有看错的地方 | 认知闸读数更新；**这层最容易被藏起来，强制作答，不许空** |

每层一行：上期读数 → 本期读数 → 位移方向 → 一句归因。归因强制三选一：**Δ事实 / Δ价格 / Δ认知**。

三层皆无位移 → 一句话收工：**"无变化，好消息。"**

### 输出四件

1. **三层位移表**（上期读数 → 本期读数 → 归因）
2. **四象限轨迹**：仪表盘图上标三点——基线点（首次快照）→ 上期点 → 本期点，连成轨迹，看它往哪个格子挪
3. **退出信号底线校验**：逐条对照**基线**退出信号表算累计位移——相邻差分抓本季新东西，累计校验防"每季位移都小、四季出了格子"的温水漂移
4. **判定一句话**：持有 / 加注 / 减仓 / 退出 / 格子迁移警告

季检报告末尾附**本期仪表读数记录**（PROTOCOL 格式），作为下季差分的锚点。

### 防锚定（内置规则，不设独立模式）

当归因出现 **Δ认知**、或**轨迹跨格**时，追加一次**新生之眼**：不看旧快照，独立重跑一遍投资阶段流程，两份读数对照——轨迹说"它挪到哪了"，新生之眼说"没看过轨迹的人会把它标在哪"，夹角大 = 有一边在自欺。平静季度不跑，省算力。

### 节奏

每季财报披露后跑一次（A 股节奏：一季报/年报 ≤4/30、半年报 ≤8/31、三季报 ≤10/31；其他市场随当地披露节奏）。可配定时任务自动触发。

输出目录：用户当前目录，或由用户参数指定。
